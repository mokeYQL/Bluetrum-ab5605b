# ESP32S3 ↔ AB5605B 串口通信协议

## 协议约定

| 参数 | 值 |
|------|------|
| 通讯方式 | 串行异步通讯 (UART) |
| 接口电平 | TTL / LVTTL (3.3V) |
| 传送方式 | 全双工 |
| 波特率 | 115200 bps |
| 数据位 | 8 bit |
| 停止位 | 1 bit |
| 校验位 | 无 (帧级校验和保证完整性) |
| 字节序 | 大端模式 (Big-Endian) |

## 数据格式

| 类型 | 起始码（uint8） | 命令（uint8） | 长度（uint16） | 数据段                     | FCS（uint8） |
| :--- | :-------------- | :------------ | :------------- | :------------------------- | :----------- |
| 描述 | 帧头0xA5        |               | 数据段的长度   | 不同命令的数据段长度会不同 | 异或校验     |

## 命令说明

| 命令   | 含义       | 方向             |
| :----- | :--------- | :--------------- |
| `0x21` | 主机->从机 | AB5605 → ESP32S3 |
| `0x20` | 从机->主机 | ESP32S3→AB5605   |

## 功能点

| 名称 | 功能 |
|------|------|
| 查询设备信息 | 获取硬件版本、软件版本、产品名 |
| 系统控制 | 复位 |
| 静音控制 | 静音/取消静音 |
| 音量级别 | 0~16 |
| 音频切换 | `0x01`:BT-A2DP, `0x02`:AUX |
| 蓝牙连接状态 | `0x00`:断开, `0x01`:连接中, `0x02`:已连接, `0x03`:播放中 |
| 语音OTA状态 | `0x00`:成功, `0x01`:失败                                 |
| LED灯效 | `0x00`:关闭, `0x01`:常亮, `0x02`:呼吸, `0x03`:音乐律动 |

## 基础协议列表

### 1.0 查询设备信息 (命令字 `0x01`)

上控(ESP32S3)查询下控(AB5605B)的产品信息。

**上控发送**（ESP32S3 → AB5605B）：

| 字段 | 内容 |
| :--- | :--- |
| 帧头 | 0xA5 |
| 命令 | 0x20 |
| 数据长度 | 0x00 00 |
| 校验和 | 从命令到数据段的异或和 |

示例：`A5 20 01 00 00 [校验]`

**下控应答**（AB5605B → ESP32S3，数据段为JSON字符串）：

| 字段 | 内容 |
| :--- | :--- |
| 帧头 | 0xA5 |
| 命令 | 0x21 |
| 数据长度 | 0x00 N |
| 数据 | JSON字符串 |
| 校验和 | 异或校验 |

```json
{"pn":"xxx","fn":"xxx","hv":"x.x.x","sv":"x.x.x"}
```
- `pn`：下控产品名称
- `fn`：下控厂家名称
- `hv`：下控硬件版本（格式 x.x.x）
- `sv`：下控软件版本（格式 x.x.x）

数据帧示例：{"pn":"BT-BOX","fn":"Bluetrum","hv":"1.0.0","sv":"1.0.0"}

### 1.1 系统控制-复位 (命令字 `0x02`)

上控向下控发送复位指令。

`A5 20 02 00 01 01 [校验]`

> 注：下控收到复位命令后立即复位，可能无法回复响应帧。

### 1.2 静音控制 (命令字 `0x03`)

上控控制下控音频输出静音/取消静音。

| 方向 | 数据 | 示例 |
| :--- | :--- | :--- |
| 上控静音 | `0x01` 静音 / `0x00` 取消 | `A5 20 03 00 01 01 [校验]` |
| 下控应答 | `0x00` | `A5 21 03 00 01 00 [校验]` |

### 1.3 音量控制 (命令字 `0x04`)

上控设置下控的音量等级，范围0~16。

`A5 20 04 00 01 [0x00~0x10] [校验]`

示例 — 设置音量12：`A5 20 04 00 01 0C [校验]`
**下控应答**：`A5 21 04 00 01 00 [校验]`

音量等级对应表：

| 数据值 | 音量 | 说明 |
| :----: | :--: | :--- |
| 0x00 | 0 | 静音 |
| 0x08 | 8 | 中等音量（默认值） |
| 0x10 | 16 | 最大音量 |

### 1.4 音频切换 (命令字 `0x05`)

上控切换下控的音频输入源。

| 方向 | 数据 | 示例 |
| :--- | :--- | :--- |
| 上控→蓝牙 | `0x01` | `A5 20 05 00 01 01 [校验]` |
| 上控→AUX | `0x02` | `A5 20 05 00 01 02 [校验]` |
| 下控应答 | `0x00` | `A5 21 05 00 01 00 [校验]` |

### 1.5 查询蓝牙连接状态 (命令字 `0x06`)

上控查询下控当前的蓝牙连接状态。

上控发送：`A5 20 06 00 00 [校验]`

下控应答数据：

| 数据值 | 状态 |
| :----: | :--- |
| 0x00 | 已断开 |
| 0x01 | 连接中 |
| 0x02 | 已连接 |
| 0x03 | 播放中（A2DP streaming） |

示例 — 已连接并播放：`A5 21 06 00 01 03 [校验]`

### 1.6 语音OTA状态上报 (命令字 `0x07`)

下控在语音OTA更新完成后主动上报结果。

示例 — OTA成功：`A5 21 07 00 01 00 [校验]`
示例 — OTA失败：`A5 21 07 00 01 01 [校验]`

### 1.7 LED灯效控制 (命令字 `0x08`)

上控设置下控的LED灯效模式。

| 方向 | 数据 | 示例 |
| :--- | :--- | :--- |
| 上控→关 | `0x00` | `A5 20 08 00 01 00 [校验]` |
| 上控→常亮 | `0x01` | `A5 20 08 00 01 01 [校验]` |
| 上控→呼吸 | `0x02` | `A5 20 08 00 01 02 [校验]` |
| 上控→律动 | `0x03` | `A5 20 08 00 01 03 [校验]` |
| 下控应答 | `0x00` | `A5 21 08 00 01 00 [校验]` |

## 校验与重传机制

### FCS (帧校验和) 算法

```
FCS = 从命令字节到数据段末尾所有字节的异或和
     = CMD ^ LEN_H ^ LEN_L ^ DATA[0] ^ DATA[1] ^ ... ^ DATA[N-1]
```

C语言实现：

```c
u8 calc_fcs(u8 cmd, u8 *data, u16 len) {
    u8 fcs = cmd ^ (len >> 8) ^ (len & 0xFF);
    for (u16 i = 0; i < len; i++)
        fcs ^= data[i];
    return fcs;
}
```

### 超时重传

| 参数 | 推荐值 | 说明 |
| :--- | :----: | :--- |
| RESP_TIMEOUT_MS | 1000 ms | 等待下控响应的超时时间 |
| RETRY_MAX | 3 次 | 最大重试次数 |
| RESET_WAIT_MS | 2000 ms | 复位后等待下控恢复正常 |

重传流程：

1. ESP32S3 发送请求帧
2. 收到有效响应 → 处理完成
3. 超过 RESP_TIMEOUT_MS 未收到响应 → 重试，超过 RETRY_MAX 则上报通信故障
