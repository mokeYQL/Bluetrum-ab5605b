# AB5605B 蓝牙音箱软件详细设计-初版

## 1 产品概述

AB560X（RISC-V 32bit, 120MHz, 512KB Flash）+ BR/EDR + BLE 双模蓝牙 + WS2812 灯带律动蓝牙音箱。

核心配置：A2DP 音乐 / AVRCP 控制 / BLE / AUX 线路输入 / WS2812 灯带 / 电池充电 / PWRKEY 软开关机 / 提示音。关闭 HFP / SPP / U盘SD卡 / 录音 / 卡拉OK / FM。蓝牙后台保活（BT_BACKSTAGE_EN）允许 AUX 模式下保持蓝牙连接。



512KB Flash 中 492KB 用于代码和只读数据，末尾约 20KB 为参数存储区（音量、蓝牙配对等）。双线读取 + 加速模式提升取指带宽。

```mermaid
flowchart LR
    subgraph "512KB Flash"
        A["0x0000 代码+只读数据 492KB"]
        B["末尾 参数存储区 ~20KB"]
    end
    A --> C["FLASH_DUAL_READ=1 双线读取"]
    A --> D["SPIFLASH_SPEED_UP_EN=1 加速"]
    B --> E[音量/蓝牙配对/用户设置]
```

## 2 系统启动

芯片上电后从 `main()` 进入 `bsp_sys_init()`，按硬件依赖顺序依次初始化配置、IO、时钟、外设，最后根据外设状态选择初始模式。

```mermaid
flowchart TD
    A[上电/复位] --> B[main]
    B --> C[bsp_sys_init]
    C --> C1[xcfg_init 读取配置]
    C1 --> C2[bsp_io_init GPIO复用]
    C2 --> C3[bsp_var_init 清零sys_cb]
    C3 --> C4[pmu_init 电源管理]
    C4 --> C5[adpll_init + set_sys_clk 120M]
    C5 --> C6[rtc_init 32K时钟]
    C6 --> C7[param_init 恢复参数]
    C7 --> C8[bt_init 蓝牙协议栈]
    C8 --> C9[dac_init 音频输出]
    C9 --> C10[mp3_res_play 开机提示音]
    C10 --> D{外设状态判断}
    D -->|PWRON_ENTER_BTMODE| E[func_cb.sta = FUNC_BT]
    D -->|LINEIN已插入| F[func_cb.sta = FUNC_AUX]
    E --> G[func_run 主循环]
    F --> G
```

## 3 任务调度

系统采用两层循环结构：外层 `func_run()` 的 `while(1)` 负责模式切换，内层每个模式自己的 `while(sta==xxx)` 处理业务逻辑。`func_cb.sta` 被修改后内层循环退出，回到外层 switch 进入新模式。

```mermaid
flowchart TD
    RUN["func_run() while1"] --> ENTER[func_enter 通用入口]
    ENTER --> STA{func_cb.sta}

    STA -->|FUNC_BT| BT[func_bt]
    STA -->|FUNC_AUX| AUX[func_aux]
    STA -->|FUNC_PWROFF| PWR[func_pwroff]
    STA -->|FUNC_NULL| EXIT[func_exit 查表轮转]

    BT --> BT_IN["内层while: process→message→display"]
    AUX --> AUX_IN["内层while: process→message→display"]

    BT_IN -->|sta改变| RUN
    AUX_IN -->|sta改变| RUN
    PWR --> RUN
    EXIT --> RUN
```

所有模式遵循统一的 enter→process→message→display→exit 五段式结构，添加新模式只需实现这五个函数并注册到 `func_sort_table`。

```mermaid
flowchart LR
    E[func_xxx_enter] --> W["while(sta==xxx)"]
    W --> P[func_xxx_process]
    P --> M[func_xxx_message]
    M --> D[func_xxx_display]
    D --> W
    W -->|sta改变| X[func_xxx_exit]
```

## 4 中断与消息驱动

5ms 定时器中断 `usr_tmr5ms_isr()` 是所有检测逻辑的触发源，按不同周期分时执行各项检测，避免 CPU 过载。

```mermaid
flowchart TD
    ISR["usr_tmr5ms_isr()"] --> A[每次5ms]
    ISR --> B["每20ms cnt%4==0"]
    ISR --> C["每50ms cnt%10==0"]
    ISR --> D["每100ms cnt%20==0"]
    ISR --> E["每1s cnt%200==0"]

    A --> A1[DAC fade淡入淡出]
    A --> A2[bsp_key_scan按键扫描]

    B --> B1[DAC DNR动态降噪]
    B --> B2[usb_detect]
    B --> B3[sd_detect]
    B --> B4[linein_detect]
    B --> B5[earphone_detect]

    C --> C1[电池ADC采样]

    D --> D1[lowpwr_tout_ticks]
    D --> D2[gui_box_isr]

    E --> E1[MSG_SYS_1S]
    E --> E2[bsp_charge_bled_ctrl]
```

中断中检测到设备状态变化后通过 `msg_enqueue()` 入队事件，主循环 `msg_dequeue()` 取出后分发到当前模式的消息处理函数，由其修改 `func_cb.sta` 触发模式切换。

```mermaid
flowchart LR
    A[5ms中断检测] -->|设备状态变化| B[msg_enqueue]
    B --> C[消息队列]
    C --> D[msg_dequeue]
    D --> E[func_xxx_message]
    E -->|EVT_LINEIN_INSERT| F["sta = FUNC_AUX"]
    E -->|EVT_LINEIN_REMOVE| G["sta = func_cb.last"]
    E -->|KLH_POWER| H["sta = FUNC_PWROFF"]
```

## 5 蓝牙模式

### 5.1 蓝牙主循环

蓝牙模式是产品核心，进入后初始化 A2DP 通道并使能 DNR，循环中依次执行公共处理、状态更新、灯带刷新、消息处理和 LED 刷新。退出时根据 BT_BACKSTAGE_EN 决定是否保持连接。

```mermaid
flowchart TD
    ENTER[func_bt_enter] --> W["while(sta==FUNC_BT)"]
    W --> P[func_bt_process]
    P --> P1[func_process 公共处理]
    P1 --> P2[func_bt_status 状态更新]
    P2 --> P3[ws2812_flush 灯带刷新]
    P3 --> M[func_bt_message 消息处理]
    M --> D[func_bt_display LED刷新]
    D --> W
    W -->|sta改变| EXIT[func_bt_exit]
    EXIT --> EXIT1{BT_BACKSTAGE_EN}
    EXIT1 -->|1 保持连接| EXIT2[静音A2DP 不断开]
    EXIT1 -->|0| EXIT3[bt_off 完全关闭]
```

### 5.2 A2DP 音频数据流

手机发送 SBC 编码音频，经基带接收、解码、SRC 转 48K、音量控制、DNR 降噪后从 DAC 输出。同时 DAC 硬件计算音频能量供 WS2812 灯效使用。

```mermaid
flowchart LR
    A[手机A2DP] -->|SBC编码| B[蓝牙基带接收]
    B --> C[SBC解码]
    C --> D[SRC采样率转换→48K]
    D --> E[音量控制AVRCP同步]
    E --> F[DAC DNR动态降噪]
    F --> G[DAC 48KHz输出]
    G --> H[功放AMP]
    G --> I[dac_pcm_pow_calc]
    I --> J[WS2812灯带能量]
```

### 5.3 蓝牙连接状态机

上电自动回连 3 次，失败则等待新设备。连接后断线重连 20 次。每个状态对应不同的 LED 闪烁模式，连接成功/断开时播放对应提示音。

```mermaid
flowchart TD
    INIT[BT_STA_INITING] --> IDLE[BT_STA_IDLE 空闲]
    IDLE -->|上电自动回连3次| SCAN[BT_STA_SCANNING]
    SCAN -->|回连成功| CONN[BT_STA_CONNECTED]
    SCAN -->|回连失败| IDLE
    IDLE -->|新设备连接| SCAN
    CONN -->|A2DP播放| PLAY[BT_STA_PLAYING]
    PLAY -->|暂停| CONN
    CONN -->|断连| IDLE
    PLAY -->|断连重连20次| IDLE

    CONN -->|LED常亮| L1[led_bt_connected]
    SCAN -->|LED闪烁| L2[led_bt_scan]
    PLAY -->|LED不同节奏| L3[led_bt_play]
```

### 5.4 蓝牙后台保活

BT_BACKSTAGE_EN=1 时，LINEIN 插入切 AUX 后不断开蓝牙，仅静音 A2DP 输出。拔出音频线后自动切回蓝牙并恢复播放，无需重新配对。

```mermaid
flowchart TD
    A[BT模式 正常播放] -->|LINEIN插入| B[EVT_LINEIN_INSERT]
    B --> C["sta = FUNC_AUX"]
    C --> D[func_bt_exit]
    D --> E{BT_BACKSTAGE_EN}
    E -->|1| F[不调用bt_off]
    F --> G[保持A2DP连接 静音输出]
    G --> H[AUX模式运行]
    H -->|LINEIN拔出| I[EVT_LINEIN_REMOVE]
    I --> J["sta = func_cb.last = FUNC_BT"]
    J --> K[func_bt_enter]
    K --> L[取消静音 恢复播放]
```

## 6 AUX 模式

### 6.1 AUX 音频路径

模拟音频从 LINEIN 引脚输入，经 SDADC 模数转换后送 DAC 输出。ADC 路径便于数字域 EQ 和录音，AUX 模式下关闭 DNR 防止截断语音开头。

```mermaid
flowchart LR
    A["LINEIN L=PB1 R=PB2"] --> B[SDADC模数转换]
    B --> C[数字音频处理]
    C --> D[DAC 48KHz输出]
    D --> E[功放AMP]
    D --> F[dac_pcm_pow_calc]
    F --> G[WS2812 AUX灯效]
```

### 6.2 AUX 主循环

进入时初始化 SDADC 并关闭 DNR，旁路蓝牙音频。循环中处理公共任务和灯带刷新，LINEIN 拔出时切回上一模式。

```mermaid
flowchart TD
    ENTER[func_aux_enter] --> E1[初始化SDADC]
    E1 --> E2[AUX_SNR_EN=0 关闭DNR]
    E2 --> E3[bt_audio_bypass 蓝牙旁路]
    E3 --> W["while(sta==FUNC_AUX)"]
    W --> P[func_aux_process]
    P --> P1[func_process 公共处理]
    P1 --> P2[ws2812_flush 灯带刷新]
    P2 --> M[func_aux_message]
    M -->|EVT_LINEIN_REMOVE| M1["sta = func_cb.last"]
    M -->|其他| M2[func_message公共处理]
    M1 --> EXIT
    M2 --> D[func_aux_display]
    D --> W
    EXIT[func_aux_exit] --> X1[func_aux_stop]
    X1 --> X2["func_cb.last = FUNC_AUX"]
```

### 6.3 LINEIN 插拔自动切换

5ms 中断中 `linein_detect()` 检测插拔状态变化，插入时切 AUX，拔出时切回上一模式（通常是蓝牙），实现即插即用。

```mermaid
flowchart TD
    A[5ms中断 linein_detect] --> B{LINEIN状态变化}
    B -->|插入| C[msg_enqueue EVT_LINEIN_INSERT]
    B -->|拔出| D[msg_enqueue EVT_LINEIN_REMOVE]
    C --> E[func_message公共处理]
    E --> F["sta = FUNC_AUX"]
    D --> G[func_aux_message]
    G --> H["sta = func_cb.last → FUNC_BT"]
```

## 7 WS2812 灯带

### 7.1 SPI 编码原理

用 SPI 硬件编码代替 GPIO 位翻转，无需关中断。SPI1 = 2.4MHz（120M/50），PE7→DIN。每个 WS2812 bit 编码为 3 个 SPI bit，40 颗 LED 共 360 字节通过 DMA 一次性发送。

```mermaid
flowchart LR
    A["WS2812 bit=0"] --> B["SPI: 100 H≈420ns"]
    C["WS2812 bit=1"] --> D["SPI: 110 H≈840ns"]
    B --> E["1 WS2812 byte → 3 SPI bytes"]
    D --> E
    E --> F["40 LED × 3色 × 3字节 = 360B"]
    F --> G["SPI1 DMA一次性发送"]
```

### 7.2 编码流程

`ws2812_encode_byte()` 将 1 字节数据逐位编码：bit=0 拼接 100，bit=1 拼接 110，8 bit 产生 24 bit SPI 值，拆分为 3 字节输出。

```mermaid
flowchart TD
    A[ws2812_encode_byte] --> B[遍历8bit 从高到低]
    B --> C{当前bit}
    C -->|0| D["拼接 100"]
    C -->|1| E["拼接 110"]
    D --> F[24bit SPI值]
    E --> F
    F --> G["out[0] = 高8位"]
    G --> H["out[1] = 中8位"]
    H --> I["out[2] = 低8位"]
```

### 7.3 灯效逻辑

`ws2812_flush()` 以 80ms 间隔在主循环调用。BT 已连接时从中心向两边扩散，AUX 模式下顺序填充，其他状态全灭。两种灯效均依赖 DAC 硬件计算的音频能量值。

```mermaid
flowchart TD
    A["ws2812_flush() 每80ms"] --> B{func_cb.sta}

    B -->|"FUNC_BT + 已连接"| C[dac_pcm_pow_calc]
    C --> D[能量→扩散层数 0~20]
    C --> E[能量→色相 蓝绿黄红紫]
    D --> F["从中心LED19/20向两边扩散"]
    E --> F

    B -->|FUNC_AUX| G[dac_pcm_pow_calc]
    G --> H[能量→亮灯数 0~40]
    H --> I["顺序填充 绿→黄渐变"]

    B -->|其他| J[全灭]

    F --> K[ws2812_encode_byte]
    I --> K
    J --> K
    K --> L[填充ws2812_spi_buf 360B]
    L --> M["SPI1 DMA发送 PE7输出"]
```

### 7.4 BT模式能量映射

BT 模式下音频能量同时映射为扩散层数和色相：层数决定从中心向外点亮多少颗灯，色相决定整体颜色倾向。音量越大扩散越远、颜色越暖。

```mermaid
flowchart LR
    subgraph 扩散层数
        E1["<300: level=0"] --> E2["<4000: level=2"]
        E2 --> E3["<8000: level=5"]
        E3 --> E4["<12000: level=8"]
        E4 --> E5["<18000: level=11"]
        E5 --> E6["<25000: level=14"]
        E6 --> E7["<35000: level=17"]
        E7 --> E8[">=35000: level=20 全亮"]
    end
```

```mermaid
flowchart LR
    subgraph 色相映射
        H1["<1200: hue=32 蓝"] --> H2["<3000: hue=64"]
        H2 --> H3["<6000: hue=96 绿"]
        H3 --> H4["<10000: hue=128"]
        H4 --> H5["<16000: hue=160 黄"]
        H5 --> H6["<25000: hue=192"]
        H6 --> H7["<38000: hue=224 红"]
        H7 --> H8[">=38000: hue=255 紫"]
    end
```

## 8 电源管理

### 8.1 电池检测与低电保护

电池电压每 50ms 通过内部 ADC 采样，低于警告值时播放低电提示音并降低音量，持续低于阈值 255 次后强制关机。低于关机阈值则立即关机。

```mermaid
flowchart TD
    A["VBAT ADC 每50ms采样"] --> B{电压值}
    B -->|">= LPWR_WARNING_VBAT"| C[正常]
    B -->|"低于警告值"| D[播放低电提示音]
    D --> E[LOWPWR_REDUCE_VOL 降低音量]
    E --> F[启动低电计数器]
    F --> G{计数到255次}
    G -->|是| H[强制关机]
    B -->|"低于 LPWR_OFF_VBAT"| H
```

### 8.2 充电管理

片上充电控制器支持三阶段充电：涓流（<2.9V 小电流保护过放电池）→ 恒流（主充电阶段）→ 恒压（电流递减至截止），完成后蓝灯常亮。

```mermaid
flowchart TD
    A[charge_dc_detect] --> B{DC插入}
    B -->|否| C[正常供电]
    B -->|是| D{电池电压}
    D -->|"< 2.9V"| E["涓流充电 TRICKLE_CURR 10~40mA"]
    D -->|"2.9V~4.2V"| F["恒流充电 CONSTANT_CURR 10~350mA"]
    D -->|"接近4.2V"| G[恒压充电 电流递减]
    E -->|"电压升至2.9V"| F
    F --> G
    G --> H{电流 < STOP_CURR}
    H -->|是| I[充电完成 蓝灯常亮]
```

### 8.3 休眠流程

无操作超时后进入蓝牙 Sniff 休眠：关闭大部分外设，时钟降至 26MHz，蓝牙保持低功耗连接。PWRKEY 或蓝牙数据可唤醒，唤醒后恢复所有外设和 120MHz 时钟。

```mermaid
flowchart TD
    A["无操作 > sleep_time"] --> B{bt_is_sleep}
    B -->|false 蓝牙连接中| C[不进休眠]
    B -->|true| D[sfunc_sleep]
    D --> D1[bt_enter_sleep 通知协议栈]
    D1 --> D2[关闭GUI/LED/DAC/DNR/SARADC]
    D2 --> D3[关闭USB 时钟120M→26M]
    D3 --> D4[关闭PLL0/PLL1/PLL2]
    D4 --> D5[保存GPIO 未用引脚→模拟输入]
    D5 --> D6[配置唤醒源 PWRKEY下降沿]
    D6 --> D7["while(bt_is_sleep) bt_sleep_proc"]
    D7 --> E{唤醒事件}
    E -->|PWRKEY按下| F[恢复时钟120M]
    E -->|蓝牙数据到达| F
    F --> G[恢复GPIO/PLL/DAC/SARADC]
    G --> H[返回正常模式]
```

### 8.4 关机流程

长按 PWRKEY 触发关机，保存参数后调用 `sfunc_pwrdown()` 关闭所有电源域并执行 `LPMCON |= BIT(0)` 进入 Power Down，CPU 停止、功耗 μA 级。PWRKEY 或 VUSB 可唤醒，唤醒等同于冷启动。

```mermaid
flowchart TD
    A["长按PWRKEY ≥ 3~6s"] --> B["KLH_POWER消息"]
    B --> C["sta = FUNC_PWROFF"]
    C --> D[func_pwroff]
    D --> D1[LED关机指示]
    D1 --> D2[播放关机提示音]
    D2 --> D3[param_sync 保存参数]
    D3 --> D4[dac_power_off]
    D4 --> D5[saradc_exit]
    D5 --> D6[sfunc_pwrdown]
    D6 --> D7[关闭PLL/26M晶振/模拟LDO]
    D7 --> D8["配置RTC唤醒 PWRKEY/VUSB"]
    D8 --> D9["LPMCON |= BIT(0) Power Down"]
    D9 --> E["CPU停止 功耗μA级"]
    E -->|PWRKEY按下/VUSB插入| F[芯片复位 冷启动]
```

## 9 按键与提示音

### 9.1 PWRKEY 按键处理

本产品仅使用 PWRKEY 一个按键，通过 RTCCON BIT19 检测。关机态短按唤醒，运行态长按关机，短按在蓝牙模式下无功能。

```mermaid
flowchart TD
    A["PWRKEY按下 RTCCON BIT19==0"] --> B{当前状态}
    B -->|关机态| C["≥500ms → 唤醒开机"]
    B -->|运行态| D{按下时长}
    D -->|"短按 <3s"| E[蓝牙模式无功能]
    D -->|"长按 ≥3~6s"| F["KLH_POWER → sta=FUNC_PWROFF"]
```

### 9.2 提示音播放

提示音以 MP3 资源存储在 Flash，播放前需旁路蓝牙音频通道，播放完恢复。仅保留开机/关机/连接/断开/低电/最大音量 6 种提示音，其余关闭避免干扰。

```mermaid
flowchart TD
    A[触发提示音事件] --> B[bt_audio_bypass 旁路蓝牙]
    B --> C["mp3_res_play Flash资源播放"]
    C --> D[bt_audio_enable 恢复蓝牙]
```

```mermaid
flowchart LR
    subgraph 已开启
        A[POWER_ON 开机]
        B[POWER_OFF 关机]
        C[BT_CONNECT 连接]
        D[BT_DISCONNECT 断开]
        E[LOW_BATTERY 低电]
        F[MAX_VOLUME 最大音量]
    end
    subgraph 已关闭
        G[BT_WAIT_CONNECT]
        H[BT_PAIR 配对]
        I[FUNC_BT 模式切换]
        J[FUNC_AUX 模式切换]
    end
```

## 10 DAC 与音频处理

### 10.1 音频处理链

所有音频经 SRC 统一到 48KHz 后，BT 模式下经 DNR 降噪、AUX 模式下绕过 DNR，最终由 DAC 输出。功放 MUTE 控制在开关机和提示音播放时避免爆音。

```mermaid
flowchart LR
    A[音频输入] --> B[SRC采样率→48K]
    B --> C[音量控制]
    C --> D{DAC_DNR_EN}
    D -->|BT模式| E[DNR动态降噪]
    D -->|AUX模式| F["DNR关闭 AUX_SNR_EN=0"]
    E --> G[DAC输出48KHz]
    F --> G
    G --> H{LOUDSPEAKER_MUTE}
    H -->|UNMUTE| I[功放输出]
    H -->|MUTE| J[静音]
```

### 10.2 MUTE 控制

开机时先 MUTE 功放等 DAC 稳定（30ms）再 UNMUTE 避免爆音；关机/休眠/提示音播放时 MUTE 功放，播放完恢复。

```mermaid
flowchart TD
    A[开机] --> B[MUTE功放]
    B --> C["DAC稳定 30ms后"]
    C --> D[UNMUTE功放]
    E[关机/休眠] --> F[MUTE功放]
    G[提示音播放前] --> H[bt_audio_bypass]
    H --> I[播放完]
    I --> J[bt_audio_enable]
```
